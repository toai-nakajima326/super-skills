# Plan Product

## Use when

- the user is shaping a new feature or product
- the request is ambiguous or overly broad
- design direction and product wedge need to be clarified

## Workflow

1. Restate the user problem in concrete terms.
2. Identify the narrowest valuable wedge.
3. Challenge weak assumptions, vague users, and generic positioning.
4. Describe what a strong first-use experience looks like.
5. Produce a scoped product plan with explicit tradeoffs.
