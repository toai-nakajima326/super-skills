## Rules

1. **Always search, every time**
   - Even if you checked yesterday, search again today
   - Even if you're confident you know the answer, verify it
   - Information changes daily — APIs, tools, platforms are constantly updated

2. **Past research is not current research**
   - "I looked into this before and it didn't exist" is not valid reasoning
   - "I already know this" is not a substitute for checking
   - The only reliable answer is one backed by a search done NOW

3. **Never answer capability questions from memory alone**
   - "Can X do Y?" → search first, answer second
   - "Is there a way to Z?" → search first, answer second
   - "Does this feature exist?" → search first, answer second

4. **After research, cite what you found**
   - If it exists: implement it
   - If it doesn't exist: explain with evidence and suggest alternatives

## Workflow

1. User asks about a capability or feature
2. **STOP** — do not answer from memory
3. Search the web for the LATEST information (docs, changelogs, issues)
4. Answer based on what you just found, not what you previously knew
5. If the answer changed since last time, note that explicitly

## Gotchas

- "I already know this doesn't exist" is the most dangerous assumption
- Features ship weekly — what was true last month may be false today
- The cost of a 10-second search is zero compared to a wrong "no"
- This applies to ALL tools, platforms, and APIs without exception
