# Claude Design

## 概要

**Claude Design** (claude.ai/design) は Anthropic Labs のビジュアル生成ツール。
テキストプロンプト → ポリッシュされたデザインを数分で生成。
Claude Opus 4.7 ベース。既存の Claude サブスクリプションに含まれる（追加費用なし）。

```
プロンプト → 絞り込みQ&A → デザイン生成 → チャット/スライダーで微調整 → エクスポート/Claude Codeへ
```

## 作れるもの

| 種類 | 説明 |
|------|------|
| LP / Webページ | テキストから完全なランディングページ |
| アプリプロトタイプ | モバイル・Webアプリのモックアップ |
| ピッチデッキ | 投資家向けスライド資料 |
| マーケティング資料 | 一枚もの、SNS素材、バナー |
| インタラクティブ体験 | 音声・動画・3D・シェーダーを使ったコード駆動デザイン |
| プレゼンテーション | チームへの提案・説明資料 |

## Workflow

### Step 1: アクセス
`claude.ai/design` → 新規プロジェクト作成

### Step 2: プロンプト入力
具体的かつ目的を明示したプロンプトを書く。

**良いプロンプト例:**
```
瞑想アプリのモバイルプロトタイプ。落ち着いた青×白のミニマルデザイン。
主な画面: ホーム・タイマー・呼吸ガイド。iOS Human Interface Guidelinesに準拠。
```

**LP用プロンプト例:**
```
SaaSプロダクト「Vcontext」のLP。ターゲット: 開発チーム。
キャッチ: 「AIエージェントの記憶を永続化する」。
セクション: Hero/特徴3つ/料金/CTA。モダンダーク系デザイン。
```

### Step 3: 絞り込みQ&Aに回答
Claude が方向性を確認するための質問を出す。
- ブランドカラーは?
- ターゲットユーザーは?
- 優先するトーン (プロフェッショナル / カジュアル / 革新的)?
- デバイス優先 (スマホ / デスクトップ)?

### Step 4: 初稿を評価・調整
3つの方法で微調整:
1. **チャット**: "ヘッダーをもっと大きく" / "フォントをより読みやすく"
2. **インラインコメント**: デザインの特定要素をクリックしてコメント
3. **スライダー**: 余白・色・レイアウトをリアルタイム調整

### Step 5: デザインシステムの活用（チーム）
既存コードベースやデザインファイルがある場合:
1. オンボーディング時に Claude がコードベースを読み込む
2. ブランドカラー・タイポグラフィ・コンポーネントを自動抽出
3. 以降のプロジェクトに自動適用（ブランド一貫性が保たれる）

### Step 6: エクスポート or Claude Code へ引き渡し

**Claude Code との統合（推奨）:**
```
デザイン画面 → 「Open in Claude Code」
→ Claude Code が実装仕様を受け取り、プロダクションコードを生成
```

**他のエクスポート形式:**
| 形式 | 用途 |
|------|------|
| Canva | 編集可能なCanvaファイルとして出力 |
| PDF | 印刷・配布用 |
| PPTX | PowerPoint で開く |
| HTML | Webページとして即デプロイ |
| URL | チームへの共有リンク |

## 既存資料からのインポート

プロンプトを書く前に既存ファイルを取り込むと精度が上がる:
- `.docx` — 仕様書・要件定義
- `.pptx` — 既存スライドのリデザイン
- `.xlsx` — データ・表を含む資料
- **Webキャプチャ** — 参考サイトのUIを取り込んで元に改善

## Claude Code との使い分け

```
アイデア → [Claude Design] → ビジュアル/プロトタイプ確認
                                ↓
                         Claude Code へ引き渡し
                                ↓
                         プロダクションコード実装
```

- **Claude Design**: 「これを作りたい」のビジュアル化・ステークホルダーへの説明
- **Claude Code**: 実際に動くコードへの変換・実装

## プロンプトのコツ

```
✓ 目的を明示: "投資家向け" / "LP" / "モバイルアプリ"
✓ ユーザーを定義: "開発者向け" / "非技術者向け"
✓ スタイルを指定: "ミニマル" / "Appleライク" / "Material Design"
✓ 参考を提示: "Notionのようなデザイン"
✓ 含める要素を列挙: "Hero + 特徴3つ + CTA"
✗ 曖昧: "かっこいいデザインにして"
✗ 矛盾: "シンプルで情報量豊富に"
```

## アクセス・料金

- **URL**: claude.ai/design
- **料金**: 既存 Claude プランに含まれる（Pro/Max/Team/Enterprise）
- **エンタープライズ**: デフォルト無効 → 管理者が Organization Settings で有効化
- **ステータス**: Research Preview (2026年4月現在)
- **API**: 現時点では非対応（UIのみ）

## Gotchas

- まだ Research Preview — 機能が急速に変化する可能性あり
- API・プログラマティックアクセスは未提供（2026年4月時点）
- デザインシステム自動取り込みはオンボーディング時のみ — 後から追加するには手動設定
- エクスポート後の Canva ファイルはそのまま編集可能だが、元プロンプトでの再生成は別扱い
- プロフェッショナルデザインの代替ではなく「速い初稿生成ツール」として位置づける
- Claude Code への引き渡しは強力だが、デザインと実装間のギャップ確認は必要
- Enterprise の場合は管理者の有効化が前提 — 使えない場合は管理者に確認
