## Rules  
1. Dynamically adjust cache sizes based on real-time system constraints and workload patterns.  
2. Prioritize tasks by allocating CPU/memory resources based on urgency, resource availability, and task criticality.  
## Workflow  
1. Monitor system resources (CPU, memory, disk I/O) and cache hit/miss rates.  
2. Use predictive models to forecast resource needs and adjust cache configurations accordingly.  
## Gotchas  
- Over-allocating resources to cache might starve other critical tasks.  
- Requires continuous monitoring to adapt to changing workloads.  
- Potential for increased latency during resource reallocation.
